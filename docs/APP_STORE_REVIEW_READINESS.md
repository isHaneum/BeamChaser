# BeamChaser App Store Review Readiness

최종 업데이트: 2026-04-18

## 1. 현재 빌드 기준 반영 상태

- Apple Sign In, Google Sign-In, Firebase 연동 플로우 사용 중
- 설정 화면에서 개인정보 처리방침, 위치 정보 정책, 건강 데이터 정책을 인앱으로 확인 가능
- 설정 화면에서 계정 삭제 진입 가능
- 계정 삭제 시 사용자 프로필, 러닝 기록, 커뮤니티 작성물, 친구 요청 데이터 정리 로직 포함
- 연락처 기반 러너 탐색 MVP 추가
	- 연락처 이메일은 기기 내에서만 매칭에 사용
	- 서버에 주소록 전체를 업로드하지 않음
- 친구 추가를 직접 추가 방식에서 요청/수락/거절 방식으로 변경
- 커뮤니티 신고 및 사용자 차단 백엔드/클라이언트 플로우 포함
- HealthKit entitlement, 위치 권한, Contacts 권한, Google URL scheme 설정 반영
- 영어 리소스 보강 진행 중

## 2. 제출 전 필수 수동 점검

### A. App Store Connect 메타데이터

- 개인정보 처리방침 URL 입력
- Support URL 입력
- 필요 시 Marketing URL 입력
- 앱 설명, 키워드, 스크린샷, 프로모션 문구에서 미완성 한글/영문 혼용 제거
- 커뮤니티, 친구 요청, 연락처 기반 탐색이 스크린샷/설명과 실제 동작이 일치하는지 확인

### B. Firebase 운영 환경

- `firestore.rules` 실제 배포
- `storage.rules` 실제 배포
- 필요한 Firestore index 생성 여부 확인
- Firebase Authentication 공급자 상태 점검
	- Apple
	- Google
- 심사용 계정 또는 로그인 없는 재현 경로를 리뷰 노트에 명시

### C. 실기기 권한/기능 검증

- iPhone에서 Apple Sign In / Google Sign-In 모두 정상 동작 확인
- Contacts 권한 허용/거부 각각에서 커뮤니티 탐색 UI가 안전하게 동작하는지 확인
- HealthKit 읽기/쓰기 권한 요청 및 저장 결과 확인
- 위치 `While Using` / `Always` 각각에서 러닝 기록 동작 확인
- BLE 장치 연결 없이도 기본 GPS 러닝 플로우가 완료되는지 확인
- BLE 장치 연결 시 레이저 제어, 캘리브레이션, 상태 표시 확인
- Apple Watch 연동 심박수/세션 동기화 확인

## 3. 현재 심사 리스크

### 1. 커뮤니티 UGC 운영 체계

신고와 차단 기능은 코드상 존재하지만, 다음 운영 항목은 App Review 대응 문서로 반드시 보완해야 합니다.

- 신고 접수 후 처리 주체와 처리 시간
- 운영 문의를 받을 외부 지원 채널
- 반복 위반자 대응 기준
- 필요 시 커뮤니티 기능을 서버 플래그로 비활성화할 수 있는 운영 계획

### 2. 개인정보 및 App Privacy 폼

App Store Connect의 App Privacy 답변은 코드와 정책 문구에 맞게 직접 작성해야 합니다.

현재 검토 대상 데이터:

- 이름 또는 닉네임
- 이메일
- 사용자 식별자
- 위치
- 건강/피트니스 데이터
- 사진
- 연락처 기반 발견에 사용되는 이메일 정보
- 사용 데이터
	- 러닝 기록
	- 커뮤니티 활동
	- 친구 요청 상태

연락처 기능은 온디바이스 매칭이라는 점을 리뷰 노트와 정책 문구에서 일관되게 설명해야 합니다.

### 3. 정책과 실제 운영 일치 여부

- 계정 삭제 요청이 운영 백업/로그 정책과 충돌하지 않는지 확인
- 신고 데이터 보관 기간 정의
- 지원 이메일 또는 지원 웹페이지 운영 주체 확정
- 커뮤니티 신고 사유와 실제 관리자 처리 분류 체계 일치 여부 확인

### 4. 영어 품질

- 주요 온보딩, 설정, 프로필, 커뮤니티 경로는 보강 중이므로 실제 영어 환경에서 문자열 누락 여부를 최종 확인해야 함
- 일부 오래된 화면은 여전히 추가 문자열 테이블 보강이 필요할 수 있음

## 4. 제출 직전 체크리스트

1. `xcodegen`으로 프로젝트 재생성
2. `BeamChaser` iOS 빌드 확인
3. 최소 1회 이상 실기기 smoke test 수행
4. 다음 경로를 직접 재현
	 - 로그인
	 - 러닝 시작/종료
	 - 기록 확인
	 - 프로필 통계 확인
	 - 커뮤니티 피드/메이트 작성
	 - 친구 요청 송신/수락/거절
	 - 연락처 허용 후 추천 러너 표시
5. 새 build number로 Archive 생성
6. Organizer에서 Validate App 실행
7. App Store Connect 업로드 후 processing 완료 확인
8. TestFlight 내부 배포로 신규 설치/업데이트 설치 모두 확인
9. 리뷰 노트 최종 작성

## 5. 리뷰 노트 핵심 포함 항목

- Apple Sign In / Google Sign-In 테스트 방법
- HealthKit이 어떤 화면에서 왜 필요한지
- 위치 권한이 러닝 기록과 백그라운드 추적에 왜 필요한지
- Contacts 권한은 친구 탐색용 선택 기능이며 주소록 전체를 서버에 업로드하지 않는다는 점
- BLE 장치 없이도 핵심 GPS 러닝 기능을 사용할 수 있다는 점
- 커뮤니티는 로그인 후 활성화되며 신고/차단 기능이 제공된다는 점

## 6. 제출 보류 권장 조건

아래 중 하나라도 미완료면 외부 심사 제출은 보류하는 편이 안전합니다.

- 개인정보 처리방침 URL 미등록
- Support URL 미등록
- Firestore / Storage Rules 미배포
- HealthKit 실기기 저장 미검증
- Contacts 권한 거부 시 플로우 미검증
- 리뷰 노트에 로그인 및 권한 재현 절차 미기재
