# BeamChaser UI Content Rules

최종 업데이트: 2026-04-18

## 1. 언어 규칙

- 앱에서 선택한 언어가 화면 텍스트를 결정한다.
- 한국어 모드에서는 브랜드명을 제외하고 가능한 한 한국어만 사용한다.
- 영어 모드에서는 사용자에게 보이는 텍스트를 영어로만 노출한다.
- 시스템 로캘에 기대는 `LocalizedStringKey` 사용을 지양하고, 앱 언어 기준 문자열 처리 함수를 우선 사용한다.

## 2. 설명 문구 규칙

- 앱 UI에 보조 설명, 장황한 안내 문구, 분위기 설명 문구를 새로 추가하지 않는다.
- 특히 공유 카드, 카드 테마, 설정 옵션에는 설명형 카피를 붙이지 않는다.
- 필요한 경우에도 한 줄 상태 표현만 허용한다.

## 3. 공유 카드 규칙

- 공유할 실제 내용은 상단 고정 영역에서 바로 확인 가능해야 한다.
- 테마 선택 영역은 접기/펼치기를 지원한다.
- 나머지 설정값은 스크롤 영역에서 조정한다.

## 4. 레이아웃 검증 규칙

- 상단 배너, 하단 고정 버튼, 페이지 인디케이터가 있는 화면은 오버레이로 덮지 말고 실제 레이아웃 흐름 안에서 배치한다.
- 모든 주요 화면은 최소 지원 iPhone 화면에서 좌우/상하 잘림이 없어야 한다.
- 텍스트, 카드, 버튼, 이미지가 safe area 밖으로 넘어가거나 다른 컴포넌트에 가려지면 배포 전 반드시 수정한다.
- `RunActiveView` 변경 시 아래 상태를 모두 검증한다.
- 포커스 화면: 상단 상태 바, 페이스 배너, 미니 지표, 거리/시간, 음악 카드, 페이지 인디케이터, 하단 컨트롤이 동시에 전부 보여야 한다.
- 스플릿 화면: 상단 배너와 하단 컨트롤이 카드 내용을 가리지 않아야 한다.
- 러닝 중 상태와 일시정지 상태를 둘 다 확인해야 한다.
- 고정 `Spacer` 높이로 가림 문제를 회피하지 않는다. `ViewThatFits`, 축소 규칙, 스크롤, 줄임표, 최소 스케일 등 적응형 레이아웃으로 해결한다.

## 5. 브랜드명 규칙

- 사용자에게 보이는 브랜드명은 항상 `BeamChaser`만 사용한다.
- 이전 브랜드명 표기는 사용자 노출 텍스트에서 금지한다.
