BeamChaser
레이저 페이스메이커와 연동하여
목표 페이스를 지면에 시각적으로 안내하는 iOS 러닝 앱
Swift / SwiftUI CoreBluetooth iOS 17+ Arduino + BLE watchOS 10+
Problem
러너가 페이스를 유지하기 어려운 이유
기존 러닝 앱은 소리나 숫자로 피드백하여 직관성이 부족함
01
시각적 기준 부재
현재 페이스가 빠른지 느린지 화면 숫자만으로는 즉각 인지하기 어려움.
달리면서 스마트폰 화면을 확인하는 행위 자체가 위험하고 번거로움.
02
사후 피드백 방식
대부분의 앱은 km 단위 스플릿 알림을 제공하나, 실시간 조정이 불가능함.
틀어진 페이스를 인지하는 시점이 늦어 목표 달성율 저하로 이어짐.
03
청각 의존성 문제
이어폰 없이는 음성 코칭이 불가능하며, 야외 소음 환경에서 알림을 놓치는 경우가 빈번함. 이어폰 착용 자체가 주변 소음 차단으로 안전 위협 요소가 됨.
04
인터벌 트레이닝 관리 난이도
다구간 인터벌 러닝 시 각 구간의 목표 페이스를 기억하고 전환하는 것은 고도의 집중을 요구함. 구간 전환 타이밍 관리가 복잡함.
Solution
레이저로 페이스를 지면에 투사
스마트폰 + 레이저 장비 + 워치가 실시간으로 연동하는 통합 러닝 시스템

Data Flow
실시간 데이터 흐름
GPS 업데이트마다 실행되는 페이스 계산 루프 및 각 서비스의 역할
1
CoreLocation — GPS 위치 업데이트
CLLocationManagerDelegate → currentLocation, totalDistanceMeters, routePoints 갱신. 정확도 20m 이하 필터 적용.
0.5~1s 주기
2
PaceMakerEngine — 레이저 갭 계산
actualDistance vs virtualLaserDistance(=목표페이스×경과시간) 비교. gapMeters, gapSeconds 산출. ±10m 기준 ahead/onPace/behind 분류.
매 GPS 이벤트
3
BLEService — 장치 명령 전송
페이스 상태 변화 감지 시 setZone(0x04) 명령. 러닝 시작 시 setPace(0x02) + startRun(0x05) 전송. CBCharacteristic.write().
상태 변화 시
4
SwiftUI View — UI 반응형 갱신
RunSessionManager, LocationService가 @Published로 상태 방출. RunActiveView가 onChange로 구독. 갭 바, 페이스 배너, MetricView 실시간 렌더링.
Combine @Published
5
WatchConnectivity — 워치 데이터 동기화
PhoneSessionManager → WCSession.sendMessage(). 속도, 페이스, 페이스 상태, 심박수를 Apple Watch로 실시간 전달. 워치 UI 갱신.
WCSession
6
RunSessionManager — 기록 저장
종료 시 RunRecord(UUID, 경로, 거리, 시간, 페이스) 생성. JSONEncoder → UserDefaults 저장. HealthKit HKWorkout 기록 완료.
종료 트리거
