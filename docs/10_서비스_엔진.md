# 서비스 엔진

## 파일 위치
- `RunBeam/Services/LocationService.swift` — GPS + 만보기 위치 추적
- `RunBeam/Services/RunSessionManager.swift` — 러닝 세션 관리
- `RunBeam/Services/PaceMakerEngine.swift` — 페이스 계산 및 레이저 Gap 제어
- `RunBeam/Services/HealthKitService.swift` — HealthKit 운동 기록
- `RunBeam/Services/AuthService.swift` — Apple 로그인 인증
- `RunBeam/Services/BackendService.swift` — 백엔드 API 통신

---

## 1. LocationService (위치 추적)

### 역할
GPS(CLLocationManager)와 만보기(CMPedometer)를 융합하여 실시간 위치·거리·속도를 제공합니다.

### 퍼블리셔
| 프로퍼티 | 타입 | 설명 |
|----------|------|------|
| `currentLocation` | `CLLocation?` | 현재 GPS 좌표 |
| `currentSpeed` | `Double` | 현재 속도 (m/s, GPS 기반) |
| `totalDistance` | `Double` | 누적 총 거리 (m) |
| `routePoints` | `[RoutePoint]` | 기록된 경로 좌표 배열 |
| `gpsSignalStrength` | `GPSSignalStrength` | GPS 신호 상태 (.none/.weak/.moderate/.strong) |
| `authorizationStatus` | `CLAuthorizationStatus` | 위치 권한 상태 |
| `isSimulatorMode` | `Bool` | 시뮬레이터 모드 여부 |
| `currentPace` | `Double` | 현재 페이스 (분/km) |
| `splitPaces` | `[Double]` | km별 스플릿 페이스 배열 |

### 핵심 메서드
| 메서드 | 설명 |
|--------|------|
| `startTracking()` | GPS + 만보기 추적 시작 |
| `stopTracking()` | 추적 종료, 경로 저장 |
| `requestPermission()` | 위치 권한 요청 |

### GPS 신호 강도 판정
```
horizontalAccuracy ≤ 10m → .strong
horizontalAccuracy ≤ 25m → .moderate
horizontalAccuracy ≤ 50m → .weak
그 외 → .none
```

### 걸음 보정 (Sensor Fusion)
- CMPedometer로 걸음 수와 보폭 거리를 별도 측정
- GPS 거리와 만보기 거리를 비중 평균으로 보정
- 터널/실내처럼 GPS 신호가 약할 때 만보기 데이터로 거리 보완
- 보폭 캘리브레이션: `RunnerProfile.strideLength` 값 사용

### 시뮬레이터 모드
- 실제 GPS가 없는 시뮬레이터 환경에서 가상 위치 생성
- 서울 한양대 근처(37.5570, 127.0440)를 기점으로 직선 이동 시뮬레이션
- 0.5초마다 좌표 업데이트, 랜덤 노이즈 추가

### 스플릿 기록
- 1km 단위로 구간 페이스 자동 기록
- `splitPaces` 배열에 분/km 단위로 추가
- 구간 시작~끝 시간과 거리로 계산

## 2. RunSessionManager (세션 관리)

### 역할
러닝 세션의 생명주기를 관리하고, LocationService/BLEService/PaceMakerEngine을 조율합니다.

### 세션 상태 머신
```
idle ──[start]──→ running ──[pause]──→ paused
 ↑                   │                   │
 └───[finish]←───────┘                   │
 └───[finish]←───────────────────────────┘
 └───[resume]←─── paused ───[resume]──→ running
```

### 퍼블리셔
| 프로퍼티 | 타입 | 설명 |
|----------|------|------|
| `state` | `RunState` | 세션 상태 (idle/running/paused/finished) |
| `elapsedTime` | `TimeInterval` | 경과 시간 (초) |
| `currentPace` | `Double` | 현재 페이스 (분/km) |
| `currentDistance` | `Double` | 현재 거리 (m) |
| `currentCalories` | `Double` | 소모 칼로리 |
| `currentCadence` | `Int` | 현재 케이던스 (spm) |
| `heartRate` | `Int` | 현재 심박수 |
| `laserGap` | `Double` | 레이저와의 거리 차이 (m) |
| `paceStatus` | `PaceStatus` | 페이스 상태 (.tooSlow/.onPace/.tooFast) |
| `goalProgress` | `Double` | 목표 진행률 (0~1) |
| `goalAchieved` | `Bool` | 목표 달성 여부 |
| `isVoiceGuideEnabled` | `Bool` | 음성 가이드 활성화 |
| `currentSplitPaces` | `[Double]` | 현재 스플릿 페이스 배열 |
| `intervals` | `[IntervalSegment]` | 인터벌 구간 목록 |
| `currentIntervalIndex` | `Int` | 현재 인터벌 인덱스 |

### 공개 API
| 메서드 | 설명 |
|--------|------|
| `startRun(target:goal:intervals:)` | 러닝 세션 시작 |
| `pauseRun()` | 일시 정지 |
| `resumeRun()` | 재개 |
| `finishRun() → RunRecord` | 세션 종료, RunRecord 반환 |
| `toggleVoiceGuide()` | 음성 가이드 토글 |

### 타이머
- 0.1초 간격 Timer로 `elapsedTime` 업데이트
- 일시정지 시 타이머 정지, 재개 시 재시작

### 칼로리 계산
```swift
calories = METs × weightKg × hours
// METs: 속도 기반 (느리게 8.0, 보통 10.0, 빠르게 12.0)
```

### 목표 판정
| 목표 타입 | 판정 기준 |
|-----------|-----------|
| `distance` | `currentDistance ≥ goalDistance` |
| `time` | `elapsedTime ≥ goalTime` |
| `pace` | 현재 페이스가 목표 페이스 이하 (5km 이상 지속) |

### 인터벌 모드
- `intervals` 배열의 각 구간(러닝/휴식)을 순서대로 실행
- 구간 완료 시 햅틱(.warning) + 음성 알림
- 모든 구간 완료 시 자동 종료 가능

### 기록 저장
- `finishRun()` 호출 시 `RunRecord` 생성
- UserDefaults에 JSON 인코딩으로 저장
- 저장 키: `"savedRunRecords"`

## 3. PaceMakerEngine (페이스 제어)

### 역할
목표 페이스 대비 현재 페이스를 비교하여 레이저 Gap(거리 차이)과 PaceStatus를 계산합니다.

### 퍼블리셔
| 프로퍼티 | 타입 | 설명 |
|----------|------|------|
| `gap` | `Double` | 레이저와의 거리 차이 (m, 양수=앞서감) |
| `paceStatus` | `PaceStatus` | .tooSlow / .onPace / .tooFast |
| `projectedDistance` | `Double` | 레이저(가상 러너) 누적 거리 (m) |

### PaceStatus 판정
```
gap > +10m  → .tooFast (파랑, 사용자가 레이저보다 앞)
gap < -10m  → .tooSlow (빨강, 사용자가 레이저보다 뒤)
|gap| ≤ 10m → .onPace  (초록, 적정 페이스)
```

### 레이저 가상 위치 계산
```swift
// 목표 속도로 시간에 비례해 레이저 누적 거리 산출
targetSpeed = 1000.0 / targetPaceSeconds  // m/s
projectedDistance = targetSpeed × elapsedTime
gap = actualDistance - projectedDistance
```

### BLE 연동
- PaceStatus 변경 시 BLEService에 `setZone()` 전송
- 레이저 색상이 실시간으로 변경됨:
  - GREEN → 초록 레이저, RED → 빨강 레이저, BLUE → 파랑 레이저

## 4. HealthKitService (건강 데이터)

### 역할
HealthKit에 운동 세션을 기록하고, 건강 데이터를 읽어옵니다.

### 퍼블리셔
| 프로퍼티 | 타입 | 설명 |
|----------|------|------|
| `isAuthorized` | `Bool` | HealthKit 권한 부여 여부 |
| `latestHeartRate` | `Double?` | 최신 심박수 |

### 공개 API
| 메서드 | 설명 |
|--------|------|
| `requestAuthorization()` | HealthKit 읽기/쓰기 권한 요청 |
| `startWorkout()` | HKWorkoutBuilder로 운동 세션 시작 |
| `endWorkout(distance:calories:)` | 운동 세션 종료, 거리/칼로리 샘플 추가 |
| `addRouteData(locations:)` | HKWorkoutRouteBuilder로 경로 좌표 추가 |
| `queryHeartRate()` | 최신 심박수 쿼리 |

### 권한 범위
| 읽기 | 쓰기 |
|------|------|
| 심박수, 걸음수, 활성 에너지 | 운동(러닝), 경로, 거리, 활성 에너지 |

### 운동 저장 흐름
```
startWorkout()
  → HKWorkoutBuilder 생성
  → beginCollection(startDate)
  ↓
러닝 중: addRouteData(locations) 반복
  ↓
endWorkout(distance, calories)
  → 거리/칼로리 샘플 추가
  → endCollection()
  → finishWorkout()
  → finishRoute()
```

## 5. AuthService (인증)

### 역할
Sign in with Apple을 처리합니다.

### 퍼블리셔
| 프로퍼티 | 타입 | 설명 |
|----------|------|------|
| `isSignedIn` | `Bool` | 로그인 여부 |
| `userName` | `String?` | 사용자 이름 |
| `userEmail` | `String?` | 이메일 |
| `userID` | `String?` | Apple User ID |

### 인증 흐름
```
SignInWithAppleButton 탭
  → ASAuthorizationAppleIDRequest
  → Apple 인증 시트
  → didCompleteAuthorization
    → userID, fullName, email 저장 (UserDefaults)
    → isSignedIn = true
```

### 자격 증명 확인
- 앱 시작 시 `ASAuthorizationAppleIDProvider.getCredentialState()` 호출
- `.authorized` → 자동 로그인 유지
- `.revoked` / `.notFound` → 로그아웃 처리
