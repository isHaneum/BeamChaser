# BeamChaser 앱스토어 출시 가이드

최종 업데이트: 2026-04-18

## 1. 목적

이 문서는 BeamChaser iOS 앱을 App Store에 제출할 때 필요한 준비 항목, 검수 포인트, 업로드 절차, 리뷰 대응 포인트를 한 번에 정리한 운영용 가이드입니다.

## 2. 출시 범위 정리

현재 앱의 핵심 기능:

- GPS 기반 러닝 기록
- Apple Health 연동
- BLE 기반 BeamChaser 장치 연동
- 프로필, 레벨, 뱃지
- 커뮤니티 피드 및 러닝 메이트 모집
- 친구 요청 기반 소셜 관계
- 연락처 기반 러너 발견 MVP

출시 설명과 스크린샷은 반드시 위 실제 기능과 일치해야 합니다.

## 3. App Store Connect 준비물

필수 메타데이터:

- 앱 이름
- 부제
- 설명
- 키워드
- 지원 URL
- 개인정보 처리방침 URL
- 카테고리
- 연령 등급
- 스크린샷

권장 추가 자료:

- 리뷰용 데모 계정 또는 재현 절차
- 외부 하드웨어가 선택 기능이라는 설명
- 커뮤니티 운영 및 신고 대응 안내

## 4. 개인정보 및 권한 정리

앱에서 검토해야 할 권한:

- 위치
- 건강 데이터
- 블루투스
- 연락처
- 사진
- 모션

심사 대응 핵심 문장:

- 위치는 러닝 경로 기록과 백그라운드 추적에 사용
- 건강 데이터는 운동 저장/읽기에 사용
- 블루투스는 BeamChaser 장치 연결에 사용
- 연락처는 선택 기능이며 친구 탐색 보조에만 사용
- 연락처 전체를 서버 주소록 동기화 용도로 저장하지 않음

## 5. App Privacy 작성 체크

App Store Connect의 App Privacy는 코드와 정책 문구를 기준으로 직접 입력해야 합니다.

검토 데이터 범주:

- 연락처 정보
- 위치
- 건강 및 피트니스
- 사진
- 사용자 콘텐츠
- 식별자
- 이메일 또는 계정 정보
- 사용 데이터

각 항목마다 아래를 정리합니다.

- 수집 여부
- 추적 여부
- 사용자와 연결되는지 여부
- 목적

## 6. 기능별 수동 QA

### 로그인

- Apple Sign In 성공
- Google Sign-In 성공
- 로그인 취소/실패 시 오류 처리

### 러닝

- 러닝 시작
- 일시정지/재개
- 종료 후 기록 저장
- 지도, 거리, 페이스, 시간 표시 확인

### HealthKit

- 읽기 권한 요청
- 쓰기 권한 요청
- 운동 저장 성공

### 커뮤니티

- 피드 작성
- 메이트 모집 작성
- 댓글 작성
- 게시글 신고
- 사용자 차단

### 친구 시스템

- 친구 요청 보내기
- 친구 요청 수락
- 친구 요청 거절
- 이미 대기 중인 요청 중복 방지

### 연락처 기반 발견

- Contacts 허용 상태에서 추천 표시
- Contacts 거부 상태에서 graceful fallback
- 설정 앱으로 이동 동선 확인

### BLE

- 장치 검색
- 연결/해제
- 상태 표시
- 캘리브레이션 전송

## 7. 빌드 및 업로드 절차

1. `xcodegen`으로 프로젝트 동기화
2. Release 설정으로 Archive 생성
3. Organizer에서 Validate App 실행
4. Validate 경고 확인 후 App Store Connect 업로드
5. Build processing 완료 대기
6. TestFlight 내부 배포 및 설치 검증
7. 메타데이터와 리뷰 노트 작성
8. 제출

## 8. 리뷰 노트에 꼭 넣을 내용

- 로그인 없이 가능한 기능과 로그인 후 필요한 기능 구분
- BLE 장치가 선택 기능이라는 점
- 연락처 권한이 선택 기능이며 온디바이스 매칭이라는 점
- HealthKit 사용 목적
- 위치 권한 사용 목적
- 계정 삭제 경로

## 9. 자주 걸리는 리젝 포인트

### 1. UGC 운영 설명 부족

대응:

- 신고/차단 기능 존재 명시
- 운영 문의 채널 명시
- 반복 위반 대응 정책 내부 정리

### 2. 권한 요청 문구와 실제 기능 불일치

대응:

- Info.plist 문구, 개인정보 처리방침, 리뷰 노트 설명을 동일한 의미로 정렬

### 3. 하드웨어 의존성 오해

대응:

- BLE 장치 없이도 핵심 러닝 흐름을 검토 가능하다고 명시

### 4. 미완성 영어 문자열 노출

대응:

- 영어 환경에서 온보딩, 설정, 프로필, 커뮤니티 주요 경로 재검수

## 10. 제출 당일 운영 순서

1. 최종 빌드 번호 확정
2. Archive 및 업로드
3. TestFlight 설치 확인
4. 앱 설명/스크린샷 최종 검토
5. 리뷰 노트 입력
6. 제출 후 processing 및 review 상태 모니터링

## 11. 권장 산출물

- 개인정보 처리방침 웹 페이지
- 지원 웹 페이지 또는 지원 이메일
- 심사 계정 또는 재현 절차 문서
- 커뮤니티 운영 정책 요약
- 내부 QA 체크리스트 결과
